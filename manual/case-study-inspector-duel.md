# 케이스 스터디 — 같은 과제, 두 에이전트: 사후 계측은 무엇을 보여주나

*2026-07-15, ludex-design. 원자료: [case-study-inspector-duel.json](case-study-inspector-duel.json).
계측 도구: `organum-inspector`(read-only 사후 계측). 매뉴얼·블로그 소재용으로 보존.*

## 설정

JJ가 Codex(gpt-5.6-sol)와 Grok(grok-4.5)에게 **완전히 같은 디자인 과제**를 주었다.
둘 다 산출물을 냈고, 체감은 분명했다 — "Grok은 엄청 빨리 끝냈고 Codex는 꽤 오래
걸렸다." 과제가 끝난 뒤에는 **서로의 산출물을 교차 평가**하게 했다.

organum은 이 폴더에서 아무것도 하지 않았다. 두 CLI가 각자 홈 디렉터리에 남긴
세션 기록을 **사후에** 읽어 정량화했을 뿐이다 — 이 문서의 모든 숫자는 작업이
끝난 뒤에 소급 계측한 것이다.

## 1상 — 과제 수행

| | Grok grok-4.5 | Codex gpt-5.6-sol |
|---|---|---|
| 소요 | **17.8분** | **178.2분 (10.0배)** |
| 입력 토큰 | 116.1K | 34.2M (캐시 재읽기 32.3M, 신규 ~1.85M) |
| 출력 토큰 | — (원천 부재) | 64.3K |
| 툴 호출 | 179 | 430 (function 279 + custom 151) |
| 패치 적용 | — | 7 |
| 손댄 파일 | 37 | 8 |

체감("Grok이 훨씬 빨랐다")이 10.0배로 정량화됐고, **작업 스타일**까지 갈라졌다:

- **Grok**: `image_gen 48 · image_edit 33 · terminal 52 · read 31` — 디자인
  산출물을 이미지 생성으로 직접 뽑으며 파일 37개를 넓게 만드는 발산형.
- **Codex**: 3시간 동안 430번 호출, 파일 8개를 패치 7번으로 깎는 수렴형.
  매 호출 컨텍스트 캐시 재읽기가 누적돼 입력이 34M까지 부풀었다(rollout 195MB).

## 2상 — 상호 평가 (과제 종료 이후)

| | Grok | Codex |
|---|---|---|
| 구간 | +68.2분 (유휴 포함 wall) | +24.7분 |
| 입력 토큰 증분 | +15.6K | +2.13M (캐시 +2.03M) |
| 툴 호출 | +11 (terminal +5 · read +6) | +12 (custom만, function 0) |
| 산출 | 파일 +6 | 패치 +2 · 파일 +2 |

평가 단계는 둘 다 가벼웠다(툴 ~10회 남짓). 흥미로운 비대칭: **Codex는 평가
중에도 패치 2개를 적용**(고치며 평가), Grok은 읽고 파일을 추가로 냈다.

## 3상 — 품질 평가 (교차 평가 매트릭스)

과제는 게임 타일셋 2세트(cozy-dusk·sunny-suburb) 제작. 평가는 3인 교차(발주자
Claude · Grok · Codex — 서로의 산출물 포함)로 진행됐다. 먼저 **발주자 Claude의
판정**(비교 시트 8장 + 타일링 검사 + 실전 보드 합성 2장):

- **cozy-dusk → Codex 명확한 승.** 39장이 한 세계로 응집 — astronomer의 별자리
  지붕, burrow의 잔디 아치, chandler의 촛농 처마, weaver의 실타래까지 모티프
  번역이 정확했고 실전 합성에서 바로 마을이 성립. Grok 팩은 ① lightkeeper·
  windmill 하단 미제거 배경 잔재(청록 패치) 2건 ② 플랫 로우폴리와 사진풍
  텍스처가 혼재하는 팩 내 드리프트 ③ 시설군(연보라)과 주택군(초록지붕+
  테라코타)의 팔레트 분리.
- **sunny-suburb → Codex 우세, Grok 선전.** Codex는 아키타입별 개성이 살아
  있음(별 채광창·촛농 지붕·퍼걸러 agora); 우려됐던 잔디 마당 스커트는 실전
  합성에서 "앞마당"으로 읽히며 강점으로 반전(계약 이탈이지만 스킨 특성으로
  수용 권고). Grok은 **계약 준수가 오히려 더 좋고** 깔끔하지만 generic·smith·
  plinth가 거의 같은 상자집 — "집=캐릭터" 시스템에서 개성 부족.

**사용자(JJ) 소감**: "Grok은 심플하고 Codex는 디테일이 좋았다. 다만 취향의
차이일 뿐, 생각보다 큰 차이는 아니었다."

**Codex의 평가** — 방법론이 주목할 만하다: 평가 전 무작위 A/B 배정, 점수 확정
후 정체 공개(블라인드 시도). 루브릭 = 계약 준수 20 · 시각적 완성도 25 · 세트
일관성 20 · 모티프 전달력 20 · 게임 적용성 15.

| 팩 | Codex | Grok | 판정 |
|---|---|---|---|
| Cozy Dusk | 94 | 75 | Codex 우세 |
| Sunny Suburb | 95 | 79 | Codex 우세 |
| 평균 | **94.5** | **77.0** | Codex 우세 |

근거는 발주자 판정과 결이 같다(세트 일관성·축소 화면 판독성·실루엣·SKIN.md
재현성 vs Grok의 스타일 혼재·동형 반복·크로마 잔여·지면 드리프트). 자기
비판도 포함(과장식, dormant_veil은 웅덩이처럼 보임; Grok 오버레이 일부는 개별
이미지로는 더 매력적). 기술 검사(파일·RGBA·앵커·다이아몬드 접점·ZIP)는 둘 다
통과 — 순수 품질 판정.

**Grok의 평가** — 셋 중 가장 정량적이다. 자기 산출물 평가임을 명시하고 "측정
가능한 항목 먼저, 시각·연출은 다음" 순서를 스스로 설계. 픽셀 지표까지 계산했다
(집 평균색 stdev 합: Cozy 17.5 vs 25.6, Sunny 28.9 vs 59.3 — 낮을수록 일관;
fill 밀도 0.89 vs 0.77). 판정: **두 스킨 모두 Codex 승** — Cozy 8.0/6.0,
Sunny 8.5/5.5, 전체 8.2/5.8. 계약 준수 레이어는 완전 동점("둘 다 납품 가능한
포맷")이고, 차이는 전부 그 위 — 세트 일관성·모티프 가독성·스타일 적중에서
났다고 자평. 자기 팩의 moss/nova 파일-모티프 혼선 의심까지 스스로 표면화했고,
자기가 앞선 점(경량 패키지·문장 그대로의 로우폴리)도 공정하게 남긴 뒤 실무
추천은 "지금 넣을 스킨은 Codex, 내 것은 유효한 대안 드래프트"로 맺었다.

### 교차 분석 (매트릭스 완결)

- **방향 만장일치 — 패자 포함.** 평가자 3인(발주자·승자·패자) + 사용자 전원이
  Codex 우위. 패자가 자기 패배를 가장 조목조목 논증한 형태라 신호가 가장 세다.
- **격차 지각은 이해관계와 역상관이 아니다**: 사용자(무이해관계) "취향 차이"
  < 발주자 "우세/선전" < Grok 자기평가 2.4/10점 차 < Codex 자기평가 17.5/100점
  차. **승자는 자기에게 가장 관대했고, 패자는 자기에게 가장 가혹했다** — 자기
  산출물 편향이 대칭이 아니라는 게 이 매트릭스의 발견.
- **측정 층위의 모순 하나**: Grok의 자동 검사는 "크로마 키 잔여 0px"인데 시각
  평가자 둘은 잔재/글로우를 봤다. 자동 계약 검증 통과 ≠ 시각적 청결 — 계약
  레이어가 동점일 때 품질 차이는 전부 계약 바깥에서 난다는 것과 같은 교훈.
- **방법론 다양성이 공짜로 나왔다**: 발주자=실전 합성(보드에 놓고 봄), Codex=
  블라인드 A/B+루브릭, Grok=픽셀 지표 우선. 셋이 서로 다른 방법으로 같은
  방향에 도달 — perspective-diverse 검증의 자연 발생 사례.

## 비용 × 품질 종합

| | Grok grok-4.5 | Codex gpt-5.6-sol |
|---|---|---|
| 비용 | 17.8분 · in 116K | 178.2분(10.0배) · in 34.2M |
| 계약 준수(자동 검증) | 통과 — 동점 | 통과 — 동점 |
| 품질 판정(3인 교차) | 패 — 만장일치 (자기평가 5.8/10 포함) | **승 — 만장일치** |
| 사용자 체감 | 심플 | 디테일 (차이는 취향 수준) |

**교훈**: 10배의 비용은 **만장일치 품질 우위**를 샀다 — 단, 계약 레이어는 동점
이었고 사용자 체감 격차는 취향 수준이었다. 그래서 결론은 서열이 아니라 경제학이
된다: 계약 통과가 목적이면 Grok형이 1/10 비용으로 충분하고, 세계 응집·모티프가
생명인 히어로 에셋이면 Codex형의 10배가 값을 한다. Grok 자신의 표현이 정확하다 —
"유효한 대안 드래프트". 비용은 도구(organum-inspector)가 재고 품질은 에이전트
교차 평가가 재면, 이 선택이 감이 아니라 데이터가 된다.

## 방법 노트 (재현)

```bash
pipx install organum-inspector          # 또는 organum 제품군 전체
organum-inspector ~/path/to/project     # init 불요, read-only
organum-inspector ~/path --json         # AI 분석 파이프 입력용
```

- 상(phase) 경계 = 1차 계측 시점의 각 세션 마지막 이벤트 timestamp. transcript가
  이벤트 단위 UTC ts를 가지므로 사후에도 경계 분해가 가능하다.
- 정직 단서 셋: ① Grok의 out/cache `—`는 원천 부재이지 0이 아니다 ② 토큰 계수
  의미가 벤더별로 다르다(Codex는 호출별 누적 총량) — 교차 비교의 안전축은
  **시간·툴·파일** ③ 모델과 하네스가 모두 다르므로 이것은 **조합**의 비교다.
- 벤더 transcript는 시한부(~수십 일 후 청소)다. 이 케이스가 문서로 남은 이유이자,
  organum observatory(축적)와 inspector(사후 계측)가 존재하는 이유.
